# User Story 001-002: User Login

**As a** registered customer,  
**I want** to log in to my online banking account,  
**so that** I can securely access my accounts and perform activities.

**Acceptance Criteria:**
- Login form with email and password.
- Secure authentication (hash verification).
- Failed logins show generic error message.
- Account lockout after 5 failed attempts.

**Definition of Done:**
- Login API implemented.
- Session management verified.
- Tests and security checks passed.
